# Design Philosophy

## Accessibility begins with personal usability

TailorTeX starts from a simple position: accessibility is most meaningful when a person can pursue what is genuinely easiest for them to use.

There is no single interface that is accessible to everyone. People differ in disability, body, perception, attention, language, assistive technology, device, environment, and research practice. A design that removes a barrier for one researcher may create a new barrier for another. Accessibility therefore cannot be reduced to one fixed layout or a checklist applied after the main design is complete.

TailorTeX is a foundation, not a claim to have produced the final universal interface. It provides a reliable document and collaboration core that researchers can reshape around themselves.

## Personal adaptation should be ordinary

A researcher should be able to change contrast, density, timing, navigation, reading order, input method, feedback, panel structure, or even the entire presentation layer without first proving that the change is appropriate for everyone.

Personal usability is not a lesser form of design. It is a legitimate design objective and often the most direct route to meaningful access. Forking and modification are therefore expected uses of TailorTeX, not signs that the project has failed to provide enough settings.

People should not need to disclose diagnoses or medical details to justify an adaptation. Describing the barrier, desired interaction, or lived experience is enough.

## A shared foundation makes individual interfaces collaborative

Complete freedom at the interface level only works when collaborators can still exchange documents and understand the same underlying state. TailorTeX keeps a shared core for document fidelity, project meaning, comments, versions, submissions, agent protocols, recovery, and privacy.

This creates a deliberate split:

- **The foundation is shared.** Changes that affect compatibility or data meaning should be coordinated through pull requests.
- **The experience is personal.** Changes that affect how a researcher perceives and operates the system may live freely in a fork.

The goal is not to make every researcher use the same interface. The goal is to let different interfaces remain capable of research and collaboration together.

## Agents should support judgment, not hide it

TailorTeX allows researchers to delegate substantial work to agents while keeping the results, evidence, differences, and relevant source materials visible in a graphical interface. Automation should reduce mechanical effort without removing the researcher's ability to inspect and direct the work.

The interface exists as a place for understanding and judgment, not merely as a set of restrictions around automation.

## Accessibility is an ongoing practice

No release can declare accessibility finished. New forks, conflicting needs, unexpected barriers, and changes in assistive technology are part of the project's normal development. The project should preserve room for disagreement and experimentation while maintaining the shared foundation needed for collaboration.
