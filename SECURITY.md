# Security and Private-Data Reports

Please do not open a public issue for a vulnerability that could expose manuscripts, credentials, private research data, or local agent access.

Until a private reporting address is published, prepare a minimal report without sensitive attachments and contact the repository owner privately through their Git hosting profile. Include the affected version, impact, reproduction conditions, and a proposed mitigation when possible.

Do not include real research data in a proof of concept. Use synthetic files and redact local paths, tokens, email addresses, participant information, and unpublished text.
